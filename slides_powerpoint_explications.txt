========================================
SLIDES POWERPOINT - EXPLICATIONS TECHNIQUES
Projet: Iran Situation Desk
========================================

========================================
SLIDE 1: ACCUEIL FRONTOFFICE (index.php)
========================================

Titre:
Page d'accueil publique

Objectif:
Presenter le site et les 3 derniers articles publies.

Fonctionnement:
- Appelle getLatestArticles(3) depuis inc/util.php
- Afiche les titre, date et description de chaque article
- Propose une barre de recherche vers search_results.php

Points techniques:
- Requete SQL: SELECT ... FROM articles ORDER BY date_creation DESC LIMIT 3
- Exclusion des articles supprimes logiquement (article_status_id <> 2)
- Liens generes dynamiquement via le slug de l'article
- SEO: titre, meta description, viewport optimise

========================================
SLIDE 2: RECHERCHE ET FILTRAGE (search_results.php)
========================================

Titre:
Page de resultats avec recherche avancee

Objectif:
Permettre aux utilisateurs de trouvez des articles par mot-cle, tri et periode.

Fonctionnement:
- Parametres GET: q (mot-cle), sort, period, date_from, date_to, page
- Appelle searchArticles(...) dans inc/util.php
- Affiche resultats pagines (6 articles par page)

Points techniques:
- Recherche texte sur: titre_h1, meta_description, contenu_html
- Tris disponibles: date croissante/decroissante, titre A-Z/Z-A
- Filtres periodes: derniers 7 jours, 30 jours, 365 jours, personnalise
- Pagination: calcul offset et limit pour SQL
- Utilise htmlspecialchars() pour securiser l'affichage

========================================
SLIDE 3: DETAIL ARTICLE (body.php)
========================================

Titre:
Page article complet avec articles connexes

Objectif:
Afficher le contenu detaille d'un article et suggerer d'autres lectures.

Fonctionnement:
- Recupere article par slug depuis POST/GET
- Appelle getArticleBySlug($_GET['id'])
- Affiche image, titre H1, contenu HTML brut
- Affiche 3 articles lies via getOtherArticles()

Points techniques:
- Contenu stocke en HTML dans contenu_html (editeur TinyMCE au backoffice)
- <?php echo $article['contenu_html']; ?> rend le HTML directement
- Image responsive avec max-width: 100%
- Header avec barre de navigation et recherche
- Liens internes vers autres articles pour SEO

========================================
SLIDE 4: CONNEXION BACKOFFICE (login.php)
========================================

Titre:
Page de connexion securisee

Objectif:
Authentifier les editeurs avant acces au backoffice.

Fonctionnement:
- Formulaire POST vers inc/login_controller.php
- Verifie identifiant et mot de passe en base
- Affiche erreurs si invalides

Points techniques:
- Session PHP: $_SESSION['user'] est definie apres authentification
- Hash des mots de passe: password_verify($password, $hash)
- Verification en 2 etapes: existence user puis match password
- Redirection vers index_back_office.php en succes
- Code d'erreur 1: identifiants invalides, code 2: pas de session

========================================
SLIDE 5: CREATION D'ARTICLE (index_back_office.php)
========================================

Titre:
Formulaire de creation d'un nouvel article

Objectif:
Permettre aux editeurs de rediger et publier un nouvel article.

Fonctionnement:
- Formulaire POST vers inc/article_controller.php?action=create
- Integration TinyMCE pour contenu riche
- Upload optionnel d'une image (hero image)
- Saisie de: titre, slug, contenu HTML, image, meta description

Points techniques:
- Verification session: redirect vers login.php si absent
- Validation: titre et slug obligatoires
- Upload image: validation type (jpg, png, gif) et taille (< 300KB)
- Creation BD: INSERT INTO articles avec statut default = 1 (publie)
- Redirect vers articles_list.php avec msg=created apres succes

========================================
SLIDE 6: GESTION DES ARTICLES (articles_list.php)
========================================

Titre:
Tableau de pilotage des articles existants

Objectif:
Voir, editer ou supprimer les articles actifs.

Fonctionnement:
- Affiche tous les articles non supprimes (status <> 2)
- Tableau avec: titre, slug, image, date, actions
- Actions: voir detail, editer, supprimer

Points techniques:
- SELECT avec ordre decroissant par date_creation
- Soft delete: UPDATE ... SET article_status_id = 2 (ne supprime pas physiquement)
- Affichage des messages de confirmation (success/error)
- Boutons d'action generes pour chaque article
- Statut compte: nombre total d'articles actifs

========================================
SLIDE 7: DETAIL ET CONTROLE (article_details.php)
========================================

Titre:
Affichage detaille d'un article + actions

Objectif:
Consulter l'article complet avant edition ou suppression.

Fonctionnement:
- Charge article par id avec verification d'existence
- Affiche contenu similaire a body.php (frontoffice)
- Proposer boutons editer et supprimer
- Panel lateral: metadonnees techniques

Points techniques:
- Protection session: redirect vers login.php si absent
- Verification id: erreur si article non trouve
- Affiche metadonnees: id, date creation, slug, presence image, taille contenu
- Liens rapides vers articles_list.php ou nouvel article

========================================
SLIDE 8: EDITION D'ARTICLE (edit_article.php)
========================================

Titre:
Formulaire modification d'un article existant

Objectif:
Permettre aux editeurs de modifier titre, contenu, image et metadonnees.

Fonctionnement:
- Charge article par id
- Formulaire POST vers inc/article_controller.php?action=update
- Image optionnelle: remplacer ou garder l'existante
- Cloud TinyMCE pour contenu riche

Points techniques:
- Verification session et existence article
- UPDATE articles WHERE id = :id
- Preservation image si fichier non envoye
- Slug unique: contrainte unique en BD (error si doublon)
- Redirect vers articles_list.php avec msg=updated

========================================
RESUME ARCHITECTURE
========================================

Stack technique:
- PHP 7+ (POO + Prepare statements)
- MySQL/MariaDB (PDO avec prepared queries)
- HTML5 + CSS3 (responsive design)
- JavaScript vanilla (TinyMCE, formulaires)
- TinyMCE 8.3.2 (editeur WYSIWYG)

Donnees principales:
- Table: articles
  * id, titre_h1, url_slug, meta_description, image_url, image_alt
  * contenu_html (stockage du contenu editeurs)
  * date_creation, article_status_id (1=publie, 2=supprime)
- Table: utilisateurs
  * identifiant, mot_de_passe (hash)

Securite:
- Sessions PHP protegees
- Prepared statements contre injection SQL
- htmlspecialchars() pour XSS
- Password hashing avec PASSWORD_DEFAULT
- Soft delete plutot que suppression physique

Flux public:
index.php -> search_results.php -> body.php

Flux admin:
login.php -> index_back_office.php -> articles_list.php -> (article_details.php | edit_article.php)

========================================
POINTS CLEFS A RETENIR
========================================

1. Double flux: PUBLIC (3 pages) + ADMIN (5 pages)
2. Authentification: toutes pages admin necessite session
3. Soft delete: articles marques deleted, jamais vraiment effaces
4. Slug unique: identifiant URL de l'article pour SEO
5. TinyMCE: stocke HTML brut, rendu direct au frontend
6. PDO + Prepared statements: securite maximale
7. Pagination: 6 articles par page en recherche
8. Image optionnelle: dimensions responsive front, validation upload back
